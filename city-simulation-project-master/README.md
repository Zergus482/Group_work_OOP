## Требования к разработке

Каждый участник проекта должен ответственно подходить к выполнению своих задач. Работа ведется в отдельных ветках, создаваемых для каждой задачи. Необходимо строго соблюдать принципы ООП и паттерн MVVM при разработке.

Код должен быть качественным и хорошо документированным. Для каждой задачи требуется создать не менее 10 сущностей. Все коммиты должны иметь четкие и понятные названия, отражающие суть изменений.

После завершения работы над задачей необходимо оформить Pull Request и пройти процедуру код-ревью. Только после одобрения другими участниками проекта изменения могут быть объединены с основной веткой.

Помните, что от качества работы каждого из нас зависит успех всего проекта. Соблюдайте установленные правила и сроки выполнения задач.
